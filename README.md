# Note2Red

这是 `note-to-red` 的个人定制版本，用于在 Obsidian 中将 Markdown 笔记导出为适合小红书等图文平台发布的图片。

## 来源与许可

本项目基于上游项目 [Yeban8090/note-to-red](https://github.com/Yeban8090/note-to-red) 定制。

上游项目采用 MIT License。当前仓库保留 MIT 许可证，并在此基础上记录个人定制改动。原项目版权归原作者所有，自定义修改由本仓库维护。

## 这个版本的定位

原作者版本提供了从 Obsidian 笔记生成小红书风格图片的基础能力。本仓库是在此基础上，为高频写作、长文分页、封面制作和内容分发流程做的个人定制版。

这个版本更关注三件事：

- 让长文自动拆成多张图，减少手动排版。
- 让代码块、表格、引用块等复杂内容尽量稳定显示。
- 让封面、导出命名、作者信息和常用设置更适合自己的发布流程。

## 相对原作者版的主要改进

### 1. 自动分页逻辑

- 不再要求必须用一级/二级标题拆分内容。
- 插件会按图片可用高度自动分页。
- 在正文中单独插入 `---` 可以手动指定换页。
- 段落、列表、代码块、表格支持更细粒度的分页处理，减少某一页空白过大或内容溢出。

### 2. 复杂内容处理

- 表格可以按行分页，并在新页保留表头。
- 代码块可以按行分页。
- 引用块中嵌套表格/代码块时，尽量避免“说明文字”和表格被拆得不连贯。
- 表格字号、行高、内边距做了压缩，适合小红书图片里的有限空间。

### 3. 溢出策略

顶部工具栏新增三种处理方式：

- `切分`：尽量把内容拆到下一页。
- `缩放`：对超高代码块或表格做整体缩放。
- `换页`：保留块级内容完整性，直接放到下一页。

### 4. 导出体验

- 当前页导出：`{笔记名}_第01页_YYYYMMDD.png`
- 全部页导出：`{笔记名}_全部N页_YYYYMMDD.zip`
- 长图导出：`{笔记名}_长图_N页_YYYYMMDD.png`
- 支持封面页命名为 `{笔记名}_封面_YYYYMMDD.png`。

### 5. 封面与 Gemini

- 支持启用第一页封面。
- 支持手动上传封面图。
- 支持调用 Gemini 生成封面图。
- 支持封面图缓存，已生成的封面会保存在库内附件目录中。
- 支持封面提示词管理，可以保存自定义提示词，也可以恢复默认预设。
- 支持两种封面布局：图片在上、标题在上。

### 6. 界面与排版定制

- 顶部提供主题、字体、字号快捷控制。
- 支持快速显示/隐藏页眉和页脚，让正文区域更大。
- 支持全局文字颜色设置。
- 作者介绍页已改为本仓库维护者的信息，并移除打赏二维码。

## 目录说明

- `manifest.json`：Obsidian 插件清单。
- `main.js`：当前可运行的插件主文件，由 `sourcecode/` 构建生成。
- `styles.css`：当前可运行的插件样式，由 `sourcecode/` 构建生成。
- `CHANGELOG.md`：改动记录。
- `NOTES.md`：维护笔记。
- `ROADMAP.md`：后续计划。
- `sourcecode/`：后续源码维护目录，建立后日常只改这里的源码。

## 源码构建

`sourcecode/` 已放入上游源码工程，作为后续迁移定制逻辑的基础。

```bash
cd sourcecode
npm ci
npm run build
```

普通 `npm run build` 只输出到 `sourcecode/dist/`，用于安全验证，不会覆盖 Obsidian 当前正在运行的插件。

只有确认源码已经迁移完成并测试通过后，才执行：

```bash
npm run build:runtime
```

这个命令会输出到 `.obsidian/plugins/obsidian-note2red/`，覆盖 Obsidian 实际加载的 `main.js` 和 `styles.css`。

## 使用提示

- 打开一篇 Markdown 笔记后，通过命令或侧边栏图标打开 Note2Red 预览。
- 普通内容会自动按页面高度分页。
- 想强制换页时，在正文中单独写一行 `---`。
- 如果表格或代码块太高，可以切换顶部的溢出策略。
- 如果开启封面，第一页会作为封面，后续页才是正文分页。
- 修改源码后需要执行 `npm run build:runtime`，Obsidian 里重新加载插件后才会看到效果。

## 注意事项

- Gemini API Key 保存在 Obsidian 运行目录的 `data.json` 中，不要提交到公开仓库。
- `sourcecode/dist/` 只是本地构建验证产物，真正运行的是 `.obsidian/plugins/obsidian-note2red/` 下的 `main.js` 和 `styles.css`。
- 本仓库是个人定制版，不承诺与上游版本保持同步；如需合并上游功能，应先阅读 `NOTES.md` 和 `CHANGELOG.md`。

## 维护原则

日常只改 `sourcecode/` 里的源码。外层 `main.js` 和 `styles.css` 应由构建流程生成，不建议直接手改打包产物。
